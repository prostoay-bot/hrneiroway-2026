# HRNEIROWAY 2026 — Animation System

## 1. Motion Vision

Анимация на сайте HRNEIROWAY 2026 должна создавать ощущение премиального editorial-путешествия.

Она не должна быть “эффектом ради эффекта”.

Главная задача motion:

* вести взгляд пользователя;
* соединять секции;
* усиливать метафору “хаос → структура → материал → результат”;
* делать сайт живым;
* помогать понять, что материалы создаются, раскладываются, открываются и становятся понятными.

Анимация должна ощущаться:

* мягкой;
* спокойной;
* дорогой;
* ненавязчивой;
* функциональной;
* лёгкой по производительности.

## 2. Motion Principles

### 1. Meaningful Motion

Каждая анимация должна иметь смысл.

Разрешены анимации, которые:

* ведут взгляд;
* показывают связь между блоками;
* объясняют структуру;
* помогают понять интерактив;
* усиливают ощущение рабочего пространства.

Запрещены анимации, которые:

* просто двигают элементы без смысла;
* отвлекают от текста;
* замедляют чтение;
* выглядят как развлечение;
* перегружают страницу.

### 2. Premium Calm

Движение должно быть медленным, плавным и уверенным.

Избегать:

* резких прыжков;
* bounce-эффектов;
* вращений на 360 градусов;
* чрезмерного parallax;
* быстрых вспышек;
* хаотичного появления элементов;
* постоянного движения, которое раздражает.

### 3. Editorial Flow

Анимация должна поддерживать ощущение непрерывной истории:

* документ может слегка уходить в следующий блок;
* линия может вести пользователя вниз;
* карточки могут раскладываться;
* закладка может появляться как маркер важного;
* стрелка может мягко показывать направление.

### 4. Performance First

Если анимация ухудшает скорость, читаемость или mobile UX — её нужно упростить или убрать.

Не использовать тяжёлые JS-анимации без необходимости.

Приоритет:

1. CSS transitions.
2. CSS keyframes.
3. Intersection Observer для появления элементов.
4. JS только для простых scroll-triggered эффектов.
5. Сложные библиотеки не подключать без отдельного согласования.

## 3. Timing and Easing

### Default Timing

Использовать спокойные длительности:

* micro interactions: 180–260ms;
* hover effects: 220–320ms;
* card reveal: 420–700ms;
* section reveal: 600–900ms;
* decorative line draw: 900–1400ms;
* hero entrance: 800–1200ms.

### Easing

Основной easing:

* cubic-bezier(0.22, 1, 0.36, 1)

Дополнительный спокойный easing:

* cubic-bezier(0.16, 1, 0.3, 1)

Для hover:

* ease-out.

Не использовать резкий linear для крупных движений.

## 4. Core Animation Patterns

### 1. Hero Entrance

При загрузке первого экрана:

* заголовок появляется мягко снизу на 16–24px;
* подзаголовок появляется чуть позже;
* CTA появляется после текста;
* визуальные элементы рабочего стола появляются с лёгким stagger;
* без резких scale-эффектов.

Длительность:

* 800–1200ms.

Цель:

* создать ощущение спокойного начала, а не рекламного баннера.

### 2. Living Scroll Arrow

На hero должна быть живая стрелка вниз.

Варианты:

* мягкое движение вниз на 8–12px;
* лёгкое исчезновение/появление;
* линия, которая дорисовывается;
* стрелка, которая указывает к следующей секции.

Правила:

* стрелка не должна постоянно дёргаться;
* движение должно быть медленным;
* на mobile стрелка должна быть компактной;
* если пользователь начал скроллить, стрелка может исчезать.

### 3. Section Reveal

Секции появляются постепенно при скролле:

* opacity 0 → 1;
* translateY 20–32px → 0;
* без сильного scale;
* stagger для карточек 80–140ms.

Цель:

* дать ощущение аккуратной сборки страницы.

### 4. Cards as Documents

Карточки услуг, кейсов и полезных материалов могут вести себя как документы.

Hover:

* translateY(-4px);
* лёгкое усиление тени;
* мягкое изменение рамки;
* возможна маленькая бордовая закладка;
* без сильного наклона.

Reveal:

* карточки могут появляться как разложенные листы;
* небольшое смещение допустимо;
* на mobile смещения упрощаются.

### 5. Flowing Elements Between Sections

Некоторые декоративные элементы могут визуально связывать блоки:

* тонкая линия;
* угол документа;
* бордовая закладка;
* оливковая веточка;
* стрелка;
* карточка, частично заходящая в следующий блок.

Правила:

* использовать не более 1–2 таких элементов на экран;
* не перекрывать текст;
* не ломать адаптивность;
* на mobile упрощать или скрывать.

### 6. Burgundy Bookmark Motion

Бордовая закладка — фирменный motion-элемент.

Возможные действия:

* мягко появляется сбоку карточки;
* выезжает на 6–10px при hover;
* отмечает активный раздел;
* появляется рядом с CTA;
* может служить визуальным якорем между секциями.

Правила:

* не использовать слишком часто;
* не делать анимацию резкой;
* не превращать в декоративный шум.

### 7. Olive Branch Motion

Оливковая веточка — мягкий символ роста.

Возможные действия:

* лёгкое появление при скролле;
* subtle rotate 1–2 degrees;
* мягкий fade-in;
* появление рядом с “Обо мне”, “Полезное”, кейсами.

Запрещено:

* заставлять веточку постоянно качаться;
* использовать как активный UI-элемент;
* делать её слишком большой.

### 8. Footer CTA Motion

Footer должен иметь сильное, но спокойное появление.

Рекомендации:

* крупный заголовок появляется мягко;
* CTA-кнопка появляется после текста;
* стрелка в кнопке может двигаться вправо при hover;
* название HRNEIROWAY может появляться с лёгким reveal.

Footer не должен быть перегружен множеством движущихся деталей.

## 5. Page Transitions

Для статического сайта page transitions не обязательны.

Если будут реализованы:

* fade-out/fade-in 200–300ms;
* без сложных full-screen preloader;
* без долгого ожидания;
* не мешать навигации браузера.

Приоритет — быстрая загрузка страницы.

## 6. Scroll-Based Motion

Разрешены лёгкие scroll-triggered эффекты:

* reveal sections;
* stagger cards;
* subtle parallax 5–12px для декоративных элементов;
* line draw;
* pinned complex animations не использовать на первом этапе.

Не использовать:

* тяжёлый parallax;
* scroll-jacking;
* блокировку обычного скролла;
* движение, которое мешает читать текст.

## 7. Interaction Feedback

### Links

Hover:

* цвет #660810 → #4C050C;
* возможное подчёркивание #9CAD60;
* transition 180–240ms.

### Buttons

Primary button:

* background #660810;
* hover #4C050C;
* стрелка может смещаться вправо на 4–6px;
* transition 220–300ms.

Secondary button:

* border #660810;
* hover background #660810;
* hover text #FFFCEF.

### Cards

Hover:

* translateY(-4px);
* shadow slightly stronger;
* border color becomes more visible;
* bookmark/tab may move slightly.

### Forms

Focus:

* border #660810;
* subtle shadow;
* clear visible focus.

## 8. Reduced Motion and Accessibility

Обязательно поддерживать prefers-reduced-motion.

Если пользователь предпочитает уменьшенное движение:

* отключить scroll reveal;
* отключить parallax;
* отключить looping arrows;
* оставить только простые hover/focus состояния;
* все элементы должны быть видны без анимации.

Анимация не должна быть единственным способом понять смысл.

## 9. Performance Rules

Не использовать:

* тяжёлые animation libraries без согласования;
* большие видео в hero;
* canvas/WebGL без необходимости;
* бесконечные loop-анимации на многих элементах;
* постоянные transform-эффекты на большом количестве объектов.

Использовать:

* transform и opacity;
* will-change только точечно;
* Intersection Observer;
* CSS transitions;
* lazy loading для изображений;
* оптимизированные изображения.

## 10. Motion Anti-Patterns

Запрещено:

* bounce-анимации;
* резкие прыжки;
* чрезмерный parallax;
* анимации, которые мешают читать;
* вращение карточек на 360 градусов;
* бесконечно движущиеся элементы;
* тяжелые видеозаставки;
* прелоадер, который задерживает доступ к контенту;
* анимация ради “вау”, если она не помогает смыслу;
* движение текста во время чтения.

## 11. Implementation Notes

Когда начнётся разработка:

* создать базовые CSS-классы для reveal-анимаций;
* добавить JS только для Intersection Observer;
* все анимации держать в одном CSS/JS-слое, а не размазывать по проекту;
* использовать data-атрибуты для элементов, которые должны появляться при скролле;
* не добавлять анимацию к каждому элементу автоматически;
* тестировать на mobile.
