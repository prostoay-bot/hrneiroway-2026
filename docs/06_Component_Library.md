# HRNEIROWAY 2026 — Component Library

## 1. Component Principles

Компоненты сайта должны поддерживать концепцию HR Editorial Workspace.

Они должны выглядеть как элементы премиального рабочего пространства:

* документы;
* карточки;
* папки;
* заметки;
* закладки;
* фрагменты материалов;
* редакционные блоки;
* аккуратные CTA;
* реальные визуальные доказательства.

Главные принципы:

1. Компонент должен решать UX-задачу, а не просто украшать.
2. Один и тот же компонент должен переиспользоваться на разных страницах.
3. Компоненты должны быть адаптивными.
4. Компоненты должны поддерживать SEO и доступность.
5. Компоненты должны быть визуально цельными и не выбиваться из дизайн-системы.

## 2. Layout Components

### Site Header

Назначение:

* основная навигация;
* быстрый переход к Telegram;
* поддержание ощущения премиального сайта.

Состав:

* логотип / HRNEIROWAY;
* primary navigation;
* CTA “Написать в Telegram”;
* mobile menu.

Визуал:

* светлый фон #FFFCEF;
* navy текст #13273F;
* hover #660810;
* активное состояние через бордовую закладку или тонкую линию.

Правила:

* не перегружать;
* не делать слишком высокий header;
* mobile menu должно быть простым.

### Site Footer

Назначение:

* финальный продающий экран;
* закрепление бренда;
* контакты и быстрые ссылки.

Состав:

* крупный заголовок;
* короткий текст;
* CTA “Обсудить проект”;
* контакты;
* ссылки на разделы;
* крупное название HRNEIROWAY.

Визуал:

* фон #13273F;
* текст #FFFCEF;
* CTA #660810;
* акценты #9CAD60;
* большой editorial-вид.

### Section Container

Назначение:

* единый контейнер для секций.

Параметры:

* max-width 1180–1240px;
* desktop padding 32–48px;
* mobile padding 20px;
* vertical spacing 96–140px desktop, 64–88px mobile.

### Editorial Split Section

Назначение:

* секции с текстом и визуалом.

Использование:

* hero;
* about;
* service intro;
* case intro.

Структура:

* left/right columns;
* текстовый блок;
* визуальный блок;
* декоративный элемент: линия, закладка, веточка или документ.

### Full Width CTA Section

Назначение:

* сильные призывы к действию внутри страницы.

Использование:

* после услуг;
* после кейсов;
* перед footer;
* на страницах услуг.

Визуал:

* может быть navy background;
* крупный заголовок;
* короткий текст;
* CTA;
* спокойный reveal.

## 3. Navigation Components

### Primary Navigation

Ссылки:

* Главная
* Услуги
* Кейсы
* Полезное
* Блог
* Обо мне
* Контакты

Правила:

* активная страница должна быть заметна;
* не использовать многоуровневое меню на первом этапе;
* для service pages использовать внутренние ссылки в контенте.

### Mobile Navigation

Состав:

* burger button;
* full-screen или slide panel;
* крупные ссылки;
* CTA Telegram.

Правила:

* закрывается по клику на ссылку;
* закрывается по Escape;
* доступна с клавиатуры;
* не блокирует контент некорректно.

### Breadcrumbs

Использование:

* service pages;
* blog posts;
* useful materials;
* cases.

Визуал:

* маленький текст #3A3430;
* активная ссылка #660810;
* не перегружать.

## 4. Content Components

### Hero Workspace

Назначение:

* первый экран главной страницы.

Состав:

* H1;
* подзаголовок;
* два CTA;
* премиальный визуал рабочего стола;
* живая scroll arrow;
* карточки/документы/закладка.

Обязательное ощущение:

* “здесь сложное превращают в понятное”.

### Audience Cards

Назначение:

* показать 3 ключевые аудитории.

Карточки:

1. Для HR и компаний.
2. Для бизнеса.
3. Для экспертов.

Каждая карточка:

* название;
* боль;
* решение;
* CTA.

Визуал:

* карточки как рабочие заметки или папки;
* лёгкий hover lift;
* маленькая бордовая закладка.

### Service Cards

Назначение:

* показать направления услуг.

Карточки:

* HR-материалы и адаптация;
* внутренние коммуникации;
* презентации, гайды и PDF;
* AI и автоматизация HR;
* сайты и лендинги для экспертов;
* диагностика.

Каждая карточка:

* название;
* короткое описание;
* примеры материалов;
* ссылка на service landing page.

Визуал:

* как документы/папки;
* не делать одинаковыми скучными плитками.

### Popular Solutions

Назначение:

* дать быстрые входы в самые продаваемые услуги.

Примеры:

* Создать welcome book.
* Оформить инструкцию.
* Сделать HR-презентацию.
* Упаковать экспертный гайд.
* Настроить AI-помощника.
* Получить диагностику.

Формат:

* список карточек;
* CTA внутри каждой.

### Proof / Why It Works

Назначение:

* объяснить отличие HRNEIROWAY.

Блоки:

* HR-опыт;
* структура;
* человеческий текст;
* визуальная упаковка;
* AI как инструмент.

Визуал:

* editorial grid;
* короткие тезисы;
* фрагменты материалов.

### Case Preview Card

Назначение:

* превью кейса.

Состав:

* категория;
* задача;
* что сделали;
* мини “было/стало”;
* CTA.

Визуал:

* папка клиента;
* фрагменты документа;
* не просто картинка.

### Useful Material Card

Назначение:

* карточка чек-листа, шаблона, промпта, мини-инструмента или статьи.

Состав:

* тип материала;
* название;
* польза;
* формат;
* CTA.

Визуал:

* библиотечная карточка;
* метка #9CAD60;
* бордовый CTA.

### Testimonial Card

Назначение:

* отзыв.

Состав:

* короткая цитата;
* роль клиента;
* тип проекта;
* можно без имени, если нужно.

Визуал:

* мягкая карточка #E9D4C3;
* декоративная кавычка;
* спокойная типографика.

### About Snapshot

Назначение:

* короткий блок о Марине.

Состав:

* фото;
* 3–5 фактов;
* позиционирующий текст;
* CTA.

Визуал:

* рабочий стол;
* заметки;
* фрагменты материалов;
* без пафоса.

## 5. Service Page Components

### Service Hero

Состав:

* название услуги;
* короткое объяснение;
* для кого;
* 1–2 CTA;
* visual proof: документ, макет, фото или интерфейс.

### Problem Section

Состав:

* список проблем;
* короткие примеры;
* “узнаёте?” без манипуляции.

### What You Get

Состав:

* список результатов;
* конкретные deliverables;
* формат на выходе.

### Process Steps

Состав:

1. Разбираем задачу.
2. Собираем структуру.
3. Пишем/редактируем текст.
4. Оформляем материал.
5. Передаём итог и рекомендации.

Визуал:

* шаги как карточки/листы;
* линия между шагами.

### Before After

Назначение:

* показать разницу “до/после”.

Визуал:

* две колонки;
* документ до;
* документ после;
* короткий вывод.

### FAQ

Назначение:

* снять возражения.

Правила:

* accordion;
* короткие ответы;
* без длинных полотен.

Использование:

Роль:

* рядом с шагами;

### Paper Fragment

Роль:

* создать ощущение реального рабочего материала.

Использование:

* service pages;
* cases;
* useful.

## 8. Component States

Все интерактивные компоненты должны иметь состояния:

* default;
* hover;
* focus;
* active;
* disabled, если применимо.

Focus-состояние обязательно для:

* ссылок;
* кнопок;
* форм;
* меню;
* accordion;
* mobile navigation.

## 9. Reuse Rules

Правила переиспользования:

1. Не создавать новый компонент, если уже есть подходящий.
2. Не менять визуальный язык компонента от страницы к странице.
3. Разрешены модификаторы, но не хаотичные вариации.
4. Названия классов должны быть понятными.
5. Компоненты должны быть независимыми и переносимыми.

## 10. Component Anti-Patterns

Не использовать:

* одинаковые серые карточки без характера;
* тяжёлый glassmorphism;
* generic icons вместо смысловых визуалов;
* слишком много разных кнопок;
* разные стили карточек без системы;
* сложные hover-эффекты;
* декоративные элементы, которые мешают тексту;
* аккордеоны с огромными полотнами текста.

## 11. Implementation Notes

Когда начнётся код:

* сначала создать базовые компоненты;
* не верстать каждую страницу уникально;
* вынести общие стили в CSS;
* использовать модификаторы классов;
* не дублировать кнопки и карточки вручную без системы;
* проверять mobile сразу, а не в конце.
