# HRNEIROWAY 2026 — Design System

## 1. Design Direction

Визуальное направление сайта:

HR Editorial Workspace.

Сайт должен выглядеть как премиальное редакционное рабочее пространство, где создаются HR-материалы: welcome book, инструкции, гайды, презентации, внутренние коммуникации и AI-инструменты.

Визуальный характер:

* сливочный editorial;
* глубокий navy;
* бордовая уверенность;
* оливковые живые детали;
* премиальные фото;
* реальные документы;
* много воздуха;
* спокойная типографика;
* аккуратные микроанимации;
* ощущение системы, порядка и человеческого подхода.

Сайт не должен выглядеть как:

* типовой HR-сайт;
* IT-стартап с неоном;
* Canva-шаблон;
* салон красоты;
* wellness-бренд;
* fashion-блог;
* инфобизнес-лендинг.

## 2. Color System

Использовать только эту основную палитру.

### Core Colors

| Token                 | HEX     | Role                                               |
| --------------------- | ------- | -------------------------------------------------- |
| --color-cream         | #FFFCEF | основной фон сайта                                 |
| --color-beige         | #E9D4C3 | карточки, мягкие блоки, подложки                   |
| --color-navy          | #13273F | заголовки, меню, footer, тёмные секции             |
| --color-text          | #1E1E1E | основной текст                                     |
| --color-text-muted    | #3A3430 | вторичный текст, подписи                           |
| --color-burgundy      | #660810 | CTA, кнопки, активные ссылки, важные акценты       |
| --color-burgundy-dark | #4C050C | hover, глубокие детали, тёмные бордовые элементы   |
| --color-olive         | #9CAD60 | маркеры, стикеры, листочки, маленькие живые детали |

### Color Proportion

Рекомендуемая пропорция:

* 70% — #FFFCEF;
* 15% — #13273F;
* 10% — #660810;
* 5% — #9CAD60.

Цвет #E9D4C3 используется как вспомогательный фон для карточек и мягких секций.

### Usage Rules

#### Backgrounds

Основной фон:

* #FFFCEF.

Мягкие секции:

* #E9D4C3;
* #FFFCEF с тонкими рамками;
* тёмные секции #13273F использовать редко: footer, сильный CTA, ключевой editorial-блок.

#### Text

Основной текст:

* #1E1E1E.

Заголовки:

* #13273F.

Вторичный текст:

* #3A3430.

Не использовать чистый чёрный #000000.

#### CTA and Links

Основная кнопка:

* background #660810;
* text #FFFCEF;
* hover background #4C050C.

Вторичная кнопка:

* transparent или #FFFCEF;
* border #660810;
* text #660810;
* hover background #660810;
* hover text #FFFCEF.

Ссылки:

* #660810;
* hover #4C050C;
* декоративное подчёркивание может быть #9CAD60.

#### Olive Usage

Оливковый #9CAD60 использовать только дозированно:

* маленькие стикеры;
* бейджи;
* маркеры;
* подчёркивания;
* тонкие линии;
* декоративные листочки;
* маленькие иконки;
* отдельные слова в крупных заголовках.

Не использовать #9CAD60 для длинного текста и основных кнопок.

## 3. Typography

Типографика должна поддерживать editorial, экспертность и читаемость.

### Font Direction

Рекомендованные пары:

#### Preferred Option

Headings:

* Lora или Cormorant Garamond.

Body:

* Manrope или Onest.

Для реализации начать с безопасной пары:

* headings: Lora;
* body: Manrope.

Если шрифты подключаются через Google Fonts, подключать только нужные начертания, чтобы не ухудшать скорость.

### Typography Roles

#### H1

* font-family: Lora / Cormorant Garamond;
* color: #13273F;
* desktop: 64–84px;
* mobile: 40–48px;
* line-height: 0.95–1.05;
* font-weight: 600 или 700.

#### H2

* font-family: Lora / Cormorant Garamond;
* color: #13273F;
* desktop: 44–56px;
* mobile: 32–38px;
* line-height: 1.05–1.15;
* font-weight: 600.

#### H3

* font-family: Manrope SemiBold или Lora;
* color: #13273F или #660810;
* desktop: 24–32px;
* mobile: 22–26px;
* line-height: 1.2–1.3.

#### Hero Subtitle

* font-family: Manrope;
* color: #3A3430;
* desktop: 20–24px;
* mobile: 17–19px;
* line-height: 1.4–1.55.

#### Body Text

* font-family: Manrope;
* color: #1E1E1E;
* desktop: 17–18px;
* mobile: 16–17px;
* line-height: 1.55–1.7.

#### Captions and Labels

* font-family: Manrope Medium;
* color: #3A3430;
* size: 13–15px;
* uppercase можно использовать только для маленьких editorial-меток и не злоупотреблять.

#### Buttons

* font-family: Manrope SemiBold;
* size: 15–17px;
* letter-spacing: 0–0.02em;
* тексты кнопок короткие и конкретные.

Примеры:

* Обсудить проект
* Посмотреть услуги
* Скачать чек-лист
* Написать в Telegram
* Получить диагностику

## 4. Layout and Grid

### General Layout

Сайт должен иметь много воздуха и ощущение editorial-полотна.

Рекомендуемые ограничения:

* max-width контейнера: 1180–1240px;
* внутренние отступы desktop: 32–48px;
* mobile: 20px;
* большие вертикальные отступы секций: 96–140px desktop;
* 64–88px mobile.

### Grid

Использовать:

* 12-column grid на desktop;
* 6-column или 4-column на tablet;
* 1-column на mobile.

Карточки и документы могут слегка выходить из строгой сетки, но не должны ломать читаемость и адаптивность.

### Section Rhythm

Сайт не должен ощущаться как набор одинаковых прямоугольных блоков.

Допустимы:

* overlapping cards;
* визуальные элементы между секциями;
* плавные переходы;
* документы, которые частично заходят в следующий блок;
* декоративные линии, соединяющие разделы.

## 5. Imagery and Media

Главное правило:

Фотографии и визуальные материалы должны быть доказательством экспертности, а не украшением.

### Allowed Imagery

Использовать:

* реальные фото рабочего процесса;
* реальные или суперреалистичные фото рабочего стола;
* реальные документы;
* фрагменты welcome book;
* инструкции;
* презентации;
* скриншоты интерфейсов;
* материалы “до/после”;
* premium editorial photography;
* макро-детали: бумага, скрепка, закладка, ноутбук, карандаш, карточка.

### Forbidden Imagery

Не использовать:

* generic stock photos;
* случайные фото “люди в офисе”;
* девушка с ноутбуком без смысла;
* роботы;
* неоновые AI-иллюстрации;
* абстрактные 3D-сферы;
* мультяшные иллюстрации;
* дешёвый AI-art;
* изображения, которые не доказывают ценность услуги.

### Photo Style

Фото должны выглядеть как премиальная редакционная съёмка:

* мягкий кремовый свет;
* тёплая цветокоррекция;
* натуральные тени;
* реальное пространство;
* calm premium mood;
* высокая детализация;
* без чрезмерного HDR;
* без пластиковой AI-кожи;
* без кислотной насыщенности.

## 6. Components Visual Language

### Cards

Карточки должны выглядеть как:

* документы;
* заметки;
* папки;
* листы;
* закладки;
* фрагменты рабочих материалов.

Допустимы:

* лёгкое смещение;
* мягкие рамки;
* аккуратные тени;
* небольшие повороты;
* overlapping;
* hover lift.

Недопустимы:

* тяжёлые glassmorphism-блоки;
* яркие градиенты;
* одинаковые карточки без характера;
* грубые тени.

### Buttons

Кнопки:

* понятные;
* не слишком мелкие;
* с мягким hover;
* с возможной анимацией стрелки;
* без агрессивных эффектов.

Основной стиль:

* бордовая заливка;
* сливочный текст;
* мягкое скругление;
* спокойный hover.

### Decorative Elements

Использовать:

* бордовая закладка;
* оливковая веточка;
* тонкие линии;
* стрелки;
* скрепки;
* метки;
* номера блоков;
* small editorial stickers.

Декор должен помогать навигации и смыслу, а не просто украшать.

## 7. Footer Direction

Footer должен быть не “подвалом”, а сильным финальным CTA.

Рекомендуемый вариант:

* тёмный navy background #13273F;
* крупный editorial-заголовок;
* короткий текст;
* большая бордовая кнопка;
* контакты;
* крупный логотип/название HRNEIROWAY;
* минимум второстепенных ссылок.

Footer должен создавать ощущение завершённости и приглашения к следующему шагу.

## 8. Accessibility

Минимальные правила:

* достаточный контраст текста и фона;
* не использовать оливковый для мелкого текста на светлом фоне;
* интерактивные элементы должны быть видимыми;
* hover-состояния должны иметь фокус-состояния;
* шрифт body не меньше 16px на mobile;
* анимации должны уважать prefers-reduced-motion;
* кнопки и ссылки должны быть доступны с клавиатуры.

## 9. Implementation Notes

Когда начнётся код:

* все цвета оформить как CSS-переменные;
* все размеры и отступы оформить как токены;
* не дублировать значения вручную без необходимости;
* создать базовые utility-классы для контейнера, секций, кнопок и карточек;
* дизайн должен быть масштабируемым для многостраничного сайта.
